# 02. The First Principles

ICOS begins with first principles.

Not screens.

Not dashboards.

Not database tables.

Operations come first.

Software follows operations.

## Start From Workflows

The system must begin with what people actually do.

Who receives the patient?

Who requests documents?

Who protects referral ownership?

Who sends the hospital package?

Who follows up the MSO?

Who tracks the quote?

Who owns the next action?

Workflow reveals the architecture.

## Software Follows Operations

Software should not force the organization into artificial categories.

The platform should observe the work, preserve the logic, and reduce risk.

When the operation says a Case can have multiple hospital referrals, the software must support that.

When the operation says duplicates should be reviewed but not blocked, the software must support that.

When the operation says commission ownership can change only with a reason and audit trail, the software must support that.

## Cases Before Screens

The Case is not a page.

The Case is the operational unit.

Screens are only windows into the Case.

Lead Management, Clinical Decision, Referral Protection, Hospital Referrals, Finance, Travel, and Follow-up all become meaningful because they connect back to the Case.

## Knowledge Before Automation

Automation is useful only after the organization understands what must be preserved.

If automation removes context, it creates danger.

If automation hides accountability, it weakens trust.

If automation overwrites history, it destroys knowledge.

ICOS should automate carefully, but preserve human control where decisions are clinical, financial, legal, ethical, or commission-sensitive.

## Architecture Before Code

Code should express architecture.

Architecture should express business truth.

Business truth should come from real operations.

This order matters.

When code comes before architecture, systems become feature collections.

When architecture comes before code, systems become durable.
