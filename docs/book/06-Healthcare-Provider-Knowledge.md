# 06. Healthcare Provider Knowledge

The Healthcare Provider Network is not a hospital directory.

It is a case-driven knowledge base.

## Provider Network

Providers include more than hospitals.

They may include:

- Hospital groups.
- Hospital branches.
- Departments.
- Treatment programs.
- Physicians.
- International offices.
- Regional coordinators.
- Financial offices.
- Imaging centers.
- IVF centers.
- Rehab centers.
- Air ambulance providers.
- Hotel partners.
- Other healthcare-related providers.

## Operational Learning

Provider knowledge grows from real work.

When a Case is sent to a provider, the system should ask:

1. Is this an existing provider, contact, doctor, or program?
2. If yes, link the Case to the existing record.
3. If not, create a lightweight new provider or contact record.
4. Enrich the record gradually through future cases.

The organization should not be forced to manually create a complete provider database before operations begin.

## Regional Coordinators

Contacts may differ by region, language, nationality, and treatment type.

The same hospital group may have different coordinators for Iran, Africa, Europe, or Arabic-speaking patients.

ICOS must preserve these relationship details because they affect real outcomes.

## Knowledge Accumulation

Provider intelligence grows through:

- Hospital referrals.
- MSO requests.
- MSO responses.
- Quotes.
- Treatment outcomes.
- Coordinator notes.
- Patient feedback.
- Response time.
- Case completion.
- Commission experience.

Each Case makes the Provider Network smarter.

Over time, the system should help recommend the best provider based on accumulated operational evidence.
